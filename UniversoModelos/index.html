1.1.1 Bases de Datos Documentales (Document Stores)

Almacenan y consultan datos en forma de documentos, típicamente en formatos como JSON o BSON. Cada documento es una estructura de datos autocontenida y jerárquica que puede anidar otros documentos o listas. Este modelo se asemeja mucho a los objetos que los desarrolladores utilizan en el código de la aplicación, lo que reduce la "impedancia de desajuste objeto-relacional" (Saladage & Fowler, 2012). Son de propósito general y extremadamente flexibles, ya que cada documento puede tener su propia estructura.
o Ejemplo líder: MongoDB (Erickson, 2024).

1.1.2 Bases de Datos Clave-Valor (Key-Value Stores)

Son el tipo más simple de base de datos NoSQL. Almacenan datos como un diccionario o un mapa hash, donde cada dato (el "valor") está asociado a una única "clave". Los valores pueden ser desde una simple cadena de texto hasta un objeto complejo. Son increíblemente rápidas y escalables para operaciones de lectura y escritura simples, pero ofrecen capacidades de consulta limitadas, ya que generalmente solo se puede buscar por clave.
o Ejemplos: Redis (Amazon Web Services, s.f.), Amazon DynamoDB (Amazon Web Services).

1.1.3 Bases de Datos Columnares (Column-Family Stores)

A primera vista, pueden parecerse a las tablas relacionales, ya que tienen filas y columnas. Sin embargo, la diferencia fundamental es que los datos se almacenan por columnas en lugar de por filas. Esto las hace extremadamente eficientes para consultas analíticas que necesitan agregar valores sobre un gran número de filas, pero solo para un pequeño subconjunto de columnas.
o Ejemplo líder: Apache Cassandra (Apache Software Foundation, 2026).

1.1.4 Bases de Datos de Grafos (Graph Databases)

Están diseñadas específicamente para almacenar y navegar relaciones. El modelo de datos se compone de nodos (entidades) y aristas (relaciones que conectan los nodos). Tanto los nodos como las aristas pueden tener propiedades (atributos). Son la opción ideal para problemas donde las conexiones entre los datos son tan importantes como los datos mismos, como en redes sociales, sistemas de recomendación o detección de fraude.
o Ejemplo líder: Neo4j (Neo4j, 2026).
