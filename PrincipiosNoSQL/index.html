1
Modelado Embebido (Embedding)
En las bases de datos documentales, si los datos se acceden juntos, deben almacenarse juntos. En lugar de crear una tabla separada para las direcciones y hacer un JOIN, se anida el objeto de dirección dentro del documento del usuario. Esto reduce la necesidad de múltiples consultas y mejora drásticamente el rendimiento de la lectura.
2
Modelado por Referencia (Referencing)
Se utiliza cuando el modelo embebido conduciría a una duplicación excesiva de datos o a documentos demasiados grandes. En lugar de embeber el objeto completo, se almacena solo su ID como una referencia, similar a una clave foránea. Esto requiere una segunda consulta para obtener los datos referenciados, pero mantiene los documentos más pequeños y facilita la actualización de la entidad referenciada.
3
Diseño Orientado a la Consulta (Query-First Design)
Especialmente en bases de datos como Cassandra, el modelado comienza por identificar todas las consultas que la aplicación necesitará realizar. Luego, se diseñan tablas específicas para satisfacer cada una de esas consultas, incluso si eso significa duplicar los mismos datos en múltiples tablas con diferentes claves primarias. El objetivo es evitar lecturas lentas y complejas a toda costa.
