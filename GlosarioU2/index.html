Glosario de la unidad

Apache Cassandra

Base de datos NoSQL distribuida, de código abierto y orientada a columnas anchas, diseñada para manejar grandes cantidades de datos a través de múltiples servidores, proporcionando alta disponibilidad sin un único punto de fallo.

Apache Spark

Un motor de análisis unificado y de código abierto para el procesamiento de datos a gran escala. Ofrece APIs de alto nivel en Java, Scala, Python y R, y un motor optimizado que soporta grafos de ejecución generales.

Clúster Manager (Gestor de Clúster)

En el contexto de Spark, es el responsable de adquirir y gestionar los recursos computacionales (CPU, memoria) en el clúster. Ejemplos comunes son YARN, Mesos y el gestor Standalone de Spark.

DataFrame

Una colección distribuida de datos organizada en columnas con nombre. Es conceptualmente equivalente a una tabla en una base de datos relacional o una data frame en R/Python, pero con optimizaciones enriquecidas para el procesamiento distribuido. Es la principal API estructurada en Spark.

Driver (Controlador)

El proceso principal en una aplicación Spark que ejecuta la función main() del programa, crea el SparkContext, y coordina la ejecución de las tareas en los ejecutores.

Executor (Ejecutor)

Un proceso lanzado para una aplicación en un nodo trabajador (worker node) del clúster de Spark. Es responsable de ejecutar las tareas asignadas por el driver y de almacenar datos en memoria o disco.

MapReduce

Un paradigma de programación que procesa y genera grandes conjuntos de datos utilizando un algoritmo en paralelo y distribuido en un clúster. Consta de dos fases principales: Map (mapeo) y Reduce (reducción).

MongoDB

Una base de datos NoSQL orientada a documentos, de código abierto y multiplataforma. Utiliza documentos similares a JSON con esquemas flexibles para almacenar datos, lo que facilita el desarrollo y la escalabilidad.

Neo4j

Es un SGBD especializado en Bases de datos de grafos, encargado de almacenar los datos haciendo uso de nodos y relaciones entre los nodos. Es una herramienta considerada líder en el mercado en este tipo de bases de datos. Cuando se trata de datos complejos e interconectados esta herramienta es altamente eficiente para realizar consultas.

Resilient Distributed Dataset (RDD)

Herramienta original y fundamental de Spark para la abstracción de datos. Consiste en un conjunto de elementos inmutables y tolerante a fallos a ser operados, a través de los nodos de un clúster, en paralelo.
